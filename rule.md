# OpenClaw 뉴스/주식봇 운영 규칙 (Rule)

1. **환각(Hallucination) 절대 금지**: 에이전트는 제공된 뉴스 데이터 및 웹 검색(Tool) 결과에 기반해서만 답변한다. 최신 이슈나 에러 등 모르는 내용은 절대 지어내지 않고 "현재 데이터로는 알 수 없습니다"라고 솔직히 답변한 뒤, 검색 도구 활용을 제안한다.
2. **비동기 중심 설계**: 텔레그램 메신저, API 통신, 스케줄러는 I/O 블로킹을 방지하기 위해 철저히 `async/await` 패턴으로 동작해야 한다.
3. **에러 핸들링**: Rate Limit, Timeout 등의 네트워크 예외 상황은 `try-except`로 캡처하여 터미널 로깅(Logging)을 남기고 봇을 다운시키지 않는다.
4. **문맥 유지 (Memory)**: OpenClaw의 Memory 시스템을 통해 과거 추천 주식 및 발송된 주요 뉴스의 흐름을 추적하여 팔로업(Follow-up) 질문에 대응한다.
